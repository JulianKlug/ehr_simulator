# Session 01 — Data contract + repo scaffolding

**Goal:** a fresh clone can run `uv sync && uv run pytest` and see green. The 4 canonical in-memory shapes are locked with runtime-enforced pandera schemas, a synthetic dataset adapter produces conforming data, and CI runs on every PR. Every subsequent adapter has a reference to copy.

**Out of scope (later sessions):** FastAPI server, SQLite schema, HTMX templates, MIMIC adapter, Geneva adapter, Pydantic config validation (`study_config.yaml` / `questions.yaml`), CLI (Typer), structlog pipeline, DB migrations, UI.

## 2. `src/ehr_simulator/ingestion/exceptions.py`

```python
from dataclasses import dataclass, field

@dataclass(frozen=True)
class IngestionIssue:
    dataset: str
    patient_id: str | None
    row_idx: int | None
    reason: str

class AdapterError(Exception):
    """Raised when an ingestion adapter violates the canonical shape contract.

    In strict mode, raised on the first offending row. In non-strict mode,
    issues are accumulated in `IngestionResult.issues` and the adapter
    returns partial data; this class is raised only if the issues list is
    fetched via `raise_if_any()`.
    """
    def __init__(self, message: str, issues: list[IngestionIssue] | None = None) -> None:
        super().__init__(message)
        self.issues: list[IngestionIssue] = issues or []
```

Rationale: the tiered contract from autoplan D27 (strict vs. lenient). Session 1 keeps it as a data class + exception; the `ingestion_issues` SQLite table lands in a later session.

## 3. `src/ehr_simulator/ingestion/canonical.py`

Four pandera schemas defining the canonical in-memory shapes. Single public function `validate(frame, schema_name, *, strict=True)` routes to the right schema and returns the validated frame or raises `AdapterError`.

### Schema details

**`SCALAR_TS` — time-varying numerics (vitals, labs, anything else).**

| column | dtype | nullable | constraint |
|---|---|---|---|
| `patient_id` | `str` | no | non-empty |
| `t_minutes` | `float64` | no | `>= 0` |
| `variable` | `str` | no | non-empty |
| `value` | `float64` | **yes** | — (NaN is allowed; missing measurement) |
| `unit` | `str` | **yes** | — |
| `source` | `str` | no | non-empty |

No uniqueness constraint — multiple measurements at the same `t_minutes` are legitimate (e.g., two BP readings 10 seconds apart). Downstream code must handle duplicates.

**`source` semantics are adapter-specific, not schema-enforced.** Real-world source systems use composite strings (`EHR`, `EHR_pop_imputed`, `notes_locf_imputed`, `stroke_registry_pop_imputed`, etc.). The canonical schema accepts any non-empty string. Imputation filtering — dropping rows whose `source` contains `"imputed"` — is an **adapter** responsibility, applied before calling `validate()`. This keeps the canonical contract free of source-system conventions.

**`ADMISSION` — static patient facts.**

| column | dtype | nullable | constraint |
|---|---|---|---|
| `patient_id` | `str` | no | non-empty |
| `field` | `str` | no | non-empty |
| `value` | `str` | **yes** | — (numeric admission values are str-coerced at adapter boundary) |

DataFrame-level check: unique on `(patient_id, field)`. One value per field per patient.

**`IMAGING` — per-timepoint imaging references.**

| column | dtype | nullable | constraint |
|---|---|---|---|
| `patient_id` | `str` | no | non-empty |
| `t_minutes` | `float64` | no | `>= 0` |
| `modality` | `str` | no | non-empty |
| `report_text` | `str` | **yes** | — |
| `image_refs` | `str` | **yes** | valid JSON (list of strings) when present |

DataFrame-level check: unique on `(patient_id, t_minutes, modality)`.

`image_refs` is stored as a JSON string (list of relative filesystem paths). Keeps the pandera contract simple and serializable; session 2+ adds the sha256/mtime tracking from autoplan D37.

**`AI_OUTPUT` — precomputed per-timepoint model output.**

| column | dtype | nullable | constraint |
|---|---|---|---|
| `patient_id` | `str` | no | non-empty |
| `t_minutes` | `float64` | no | `>= 0` |
| `model_id` | `str` | no | non-empty |
| `output_json` | `str` | no | valid JSON |

DataFrame-level check: unique on `(patient_id, t_minutes, model_id)`.

### Public API

```python
from enum import Enum
import pandas as pd
import pandera.pandas as pa

class CanonicalShape(str, Enum):
    SCALAR_TS = "scalar_ts"
    ADMISSION = "admission"
    IMAGING = "imaging"
    AI_OUTPUT = "ai_output"

SCALAR_TS_SCHEMA: pa.DataFrameSchema = ...
ADMISSION_SCHEMA: pa.DataFrameSchema = ...
IMAGING_SCHEMA: pa.DataFrameSchema = ...
AI_OUTPUT_SCHEMA: pa.DataFrameSchema = ...

SCHEMAS: dict[CanonicalShape, pa.DataFrameSchema] = {
    CanonicalShape.SCALAR_TS: SCALAR_TS_SCHEMA,
    CanonicalShape.ADMISSION: ADMISSION_SCHEMA,
    CanonicalShape.IMAGING: IMAGING_SCHEMA,
    CanonicalShape.AI_OUTPUT: AI_OUTPUT_SCHEMA,
}

def validate(
    frame: pd.DataFrame,
    shape: CanonicalShape,
    *,
    strict: bool = True,
    dataset: str = "unknown",
) -> pd.DataFrame:
    """Validate `frame` against the canonical `shape` schema.

    strict=True: raise AdapterError on the first violation (pandera lazy=False).
    strict=False: collect violations into AdapterError.issues, return the
        subset of rows that DID pass (pandera lazy=True + filter).
    """
```

Module docstring locks the canonical contract in one place and is the source of truth for the `docs/data-contract.md` generator (autoplan D50, later session).

## 4. `src/ehr_simulator/ingestion/synthetic.py`

Generator producing 3 patients in all 4 canonical shapes.

```python
from dataclasses import dataclass
import pandas as pd

@dataclass(frozen=True)
class SyntheticDataset:
    scalar_ts: pd.DataFrame      # conforms to SCALAR_TS_SCHEMA
    admission: pd.DataFrame      # conforms to ADMISSION_SCHEMA
    imaging: pd.DataFrame        # conforms to IMAGING_SCHEMA
    ai_output: pd.DataFrame      # conforms to AI_OUTPUT_SCHEMA

def load_synthetic(*, seed: int = 42) -> SyntheticDataset:
    """Return 3 deterministic synthetic patients for demos and tests.

    t=0 anchor = arbitrary (no real admission time).
    Timepoints in minutes: 0, 60, 180 for each patient.
    Calls `validate(..., strict=True)` on every frame before returning.
    """
```

### Patient shapes

| patient_id | timepoints | admission | imaging | notes |
|---|---|---|---|---|
| `synth_001` | 0, 60, 180 | age=67, sex=M, nihss_admission=8, stroke_location=left_MCA, time_of_onset_minutes=45 | CT at t=0 | baseline happy path |
| `synth_002` | 0, 60, 180 | age=74, sex=F, nihss_admission=15, stroke_location=right_MCA, time_of_onset_minutes=120 | CT at t=0 | **missing labs at t=60** (exercises empty-expected state later) |
| `synth_003` | 0, 60, 180 | age=59, sex=F, nihss_admission=3, stroke_location=no_stroke, time_of_onset_minutes=30 | no imaging | **mixed in rows with realistic composite source values like `"EHR_pop_imputed"` and `"notes_locf_imputed"` that MUST be filtered by the adapter** before validation (substring match on `"imputed"`, not equality — this is the real-world failure mode for MIMIC/Geneva) |

**Variables per timepoint:**
- Vitals (in `scalar_ts`): `hr` (bpm), `sbp` (mmHg), `dbp` (mmHg), `spo2` (%), `temp` (°C)
- Labs (in `scalar_ts`): `hgb` (g/dL), `na` (mmol/L), `cr` (mg/dL), `glucose` (mg/dL)
- AI output (in `ai_output`): `output_json = {"prob_deterioration_6h": <float>, "prob_mrs_0_2_90d": <float>}` — two made-up probabilities per timepoint, one model_id: `"demo_v0"`

All values deterministic given `seed`. Use `numpy.random.default_rng(seed)` with a tight range (HR 60-100, SBP 110-160, etc.) — physiologically plausible but NOT representative of real stroke patients.

`source` field: `"synthetic"` for all non-imputed rows in `synth_001` and `synth_002`. For `synth_003`, mix in rows with realistic composite source values — `"synthetic_pop_imputed"`, `"synthetic_locf_imputed"`, `"notes_locf_imputed"` — that the adapter must drop via substring match on `"imputed"` before returning. Shape of the poisoning: ≥3 rows across ≥2 different composite source values, guaranteeing the substring filter is exercised.

## 6. Tests

### `tests/test_canonical.py`

One test per schema for the happy path + one test per schema for a specific violation. 8 tests minimum.

1. `test_scalar_ts_valid_frame` — a small hand-built frame conforming to SCALAR_TS passes `validate(..., strict=True)`.
2. `test_scalar_ts_rejects_negative_t_minutes` — same frame but with one row having `t_minutes = -1.0` raises `AdapterError` in strict mode.
3. `test_scalar_ts_accepts_arbitrary_source_strings` — a frame with `source` values `"EHR"`, `"EHR_pop_imputed"`, `"notes_locf_imputed"`, `"stroke_registry"` all pass `validate()`. Documents the design choice that source semantics are adapter-concerns, not schema-concerns.
4. `test_admission_unique_patient_field` — two rows with the same `(patient_id, field)` raises.
5. `test_imaging_unique_patient_t_modality` — duplicate `(patient_id, t_minutes, modality)` raises.
6. `test_ai_output_requires_valid_json` — `output_json = "{not json"` raises.
7. `test_strict_false_returns_partial_frame_with_issues` — one bad row + two good rows with `strict=False` returns a 2-row frame and `AdapterError.issues` has 1 entry when fetched.
8. `test_validate_routes_by_shape_enum` — passing `CanonicalShape.ADMISSION` routes to `ADMISSION_SCHEMA`, etc.

### `tests/test_synthetic.py`

Contract tests on the `load_synthetic()` return value.

1. `test_load_synthetic_returns_all_four_shapes` — `SyntheticDataset` has non-empty DataFrames for all 4 attributes.
2. `test_load_synthetic_frames_validate_strict` — each frame passes `validate(frame, shape, strict=True)`.
3. `test_load_synthetic_drops_imputed_rows` — no row in `scalar_ts` has `source == "imputed"`.
4. `test_load_synthetic_has_three_patients` — `patient_id.unique()` equals `{"synth_001", "synth_002", "synth_003"}`.
5. `test_load_synthetic_timepoints_non_negative` — all `t_minutes >= 0` across all frames that have that column.
6. `test_load_synthetic_synth_002_missing_labs_at_60min` — exact count of lab rows at `t_minutes == 60.0` for `synth_002` is 0 (asserts the deliberate partial-state edge case exists).
7. `test_load_synthetic_deterministic` — two calls with the same seed produce identical frames (assert via `pd.testing.assert_frame_equal`).

Target local runtime: `<10s`. `pytest-xdist` via `-n auto` runs tests in parallel.

- **Module docstrings:** every module has a one-paragraph docstring. `canonical.py`'s docstring locks the canonical contract in prose and is the source of truth for the future `docs/data-contract.md` generator.

## What Session 1 does NOT lock (MIMIC / Geneva adapter notes)

Landing here so the future adapter-session implementer finds the data-specific knowledge adjacent to the canonical contract.

### Source-column → canonical-shape routing

Both `.EXAMPLE_DATA_PATHS` CSVs share the shape `relative_sample_date_hourly_cat, case_admission_id, sample_label, source, value`. Routing rules derived from the observed `source` values:

| Observed `source` | Dataset | Routes to |
|---|---|---|
| `EHR` | MIMIC + Geneva | `SCALAR_TS` (monitoring, labs, perfusion-imaging numerics like `cbf_lt_30`, `tmax_gt_6`) |
| `notes` | MIMIC | `ADMISSION` (age, sex, medhist, medications, onset time, prestroke disability) |
| `stroke_registry` | Geneva | `ADMISSION` (same fields as MIMIC's `notes`) |
| `*_imputed` (any suffix) | Both | **drop** before validation — substring match on `"imputed"` |

No dedicated `imaging` source exists in either CSV. The `IMAGING` canonical shape (report text, DICOM refs) will be empty for MIMIC/Geneva until a separate imaging source is wired in. The adapter should still emit an empty DataFrame for `IMAGING` so the canonical contract holds.

### Time-unit conversion

Real data's time column is `relative_sample_date_hourly_cat` (integer hour bucket, 0 / 1 / 2 …). Canonical is `t_minutes` (float64). Adapter converts via `t_minutes = hour_cat * 60.0`. Document in the future MIMIC adapter's module docstring.

### Inverse normalization (UI display)

Values in both CSVs are z-scored against a reference population. For the simulator to show clinician-meaningful numbers (creatinine = 1.1 mg/dL, not −0.08 σ), the adapter must apply inverse normalization.

| Dataset | Params file |
|---|---|
| MIMIC | `logs_16022026_095909/reference_population_normalisation_parameters.csv` |
| Geneva | `logs_30012026_154047/normalisation_parameters.csv` |

Both have columns `variable, original_mean, original_std`. Inverse: `raw = z * std + mean`.

**Categorical-variable gotcha.** One-hot and binary variables (`sex_male`, `medhist_*_yes`, `categorical_onset_to_admission_time_*`, `prestroke_disability_(rankin)_N`) are z-scored in the same file, so naive inversion yields floats like 0.87 rather than 0/1. The adapter must consult `logs_*/categorical_variable_encoding.csv` to identify categoricals and either threshold the inverse (≥0.5 → 1, else 0) or re-expand the one-hot group to its original label. Continuous inversion applies only to true scalars.

### AI output

AI predictions live in a **separate file** (not `preprocessed_outcomes_*.csv` — those are ground-truth labels used for training). The `AI_OUTPUT` canonical schema already matches the expected shape. Path + format to be pinned in the adapter session.

### Display-value status check

The above inverse-normalization work unblocks Phase 2 UI. Without it, clinicians would see z-scores. Not a Session 1 blocker; synthetic data generates in raw ranges directly.
